AirGap Vault Distribution is not persistent , secrets generated will not be stored and need to be imported again after a restart.

This Distribution is intended to securely generate secret as well as to sign transactions offline, but not for secret storage.

AirGap Vault Distribution should not be run on a virtual machine , if the host has network connectivity even if the connectivity has been turned off temporarily.

Make sure that the client is disconnected from any network before you generated a new secret or import an existing one.
